00

Recap: LSTM

What we have. What is still missing.

What LSTM gave us — and what it didn't

What LSTM solved
Vanishing gradients over short-to-medium sequences.

Word order respected: "not happy" ≠ "happy".

Cell state as long-term memory across dozens of steps.
LSTM seq2seq setup
Encoder: reads source, produces final hidden state.

Decoder: takes that state, generates target one token at a time.

"The cat sat on the mat" → [h₆, c₆] → "Le chat était assis sur le tapis"
What LSTM still cannot do
The encoder compresses the entire source into a single fixed-size vector — the last hidden state.

For short sentences this works. For a 30-word sentence, one 256-dim vector must encode everything.

The decoder is completely blind to intermediate encoder states.

Information from early tokens is lost before the decoder needs it.
This is the bottleneck. One vector — too much to compress into.

02

Bahdanau Attention

Bahdanau, Cho, Bengio — ICLR 2015

The attention idea

Core idea: instead of compressing the encoder into one vector, keep all encoder hidden states and let the decoder look at all of them at each step — with learned weights.
Without attention
Encoder → single context vector c
Decoder receives the same c at every step.
All decoder steps equally informed — or equally lost.
With attention
Encoder → sequence of hidden states h₁ … hₙ
At each decoder step t, compute a different context vector cₜ as a weighted sum of all encoder states.

Weights say: "when generating token t, focus on encoder positions i, j, …"
A human analogy
When translating, you do not memorise the whole source before starting.

Writing "chat" → you glance back at "cat".
Writing "tapis" → you glance back at "mat".

This selective re-reading is exactly what attention computes — automatically, from gradients.
Key property: the decoder can access any encoder state directly, regardless of distance. Long-range dependencies are no longer bottlenecked.

Bahdanau attention — full derivation

At decoder step t, decoder state is sₜ. Encoder produced n hidden states h₁ … hₙ. We compute a context vector cₜ summarising the encoder output relevant to this step.
Step 1 — Alignment scores
e_{t,i} = vᵀ · tanh( W₁·hᵢ + W₂·sₜ )
Scalar score for each encoder position i: how relevant is hᵢ to the current decoder state sₜ?
W₁, W₂, v are learned weight matrices — the attention parameters.
Step 2 — Attention weights
α_{t,i} = exp(e_{t,i}) / Σⱼ exp(e_{t,j})
Raw scores normalised to a probability distribution over encoder positions. All α sum to 1.
α_{t,i} = "how much to attend to position i at step t"
Step 3 — Context vector
cₜ = Σᵢ α_{t,i} · hᵢ
Weighted sum of all encoder hidden states. Different at every decoder step.
If α_{t,3} = 0.85 → context dominated by encoder state 3.
Step 4 — Decoder update
sₜ = LSTM( [yₜ₋₁ ; cₜ], sₜ₋₁ )
Previous output token and context vector concatenated, fed to decoder LSTM together.
Everything is differentiable. Alignment weights α are learned end-to-end via backpropagation — no hand-crafted rules.

Reading Attention Maps

What the model has learned to look at.

Attention map — EN → FR

Cell (row j, col i) = attention weight α_{t=j, i}: how much the decoder attended to source token i when generating target token j.
Diagonal = word-to-word alignment · Off-diagonal = reordering · This is what TP-06 produces.

What attention maps reveal

Alignment learning
Without any explicit word alignment supervision, the model learns cat → chat, mat → tapis, sits → est assis.

This emerges purely from the translation loss, backpropagated through the attention weights.
Reordering
EN: "black cat" (adj before noun)
FR: "chat noir" (adj after noun)

The map shows: generating "chat" → attends to "cat" (pos 3), generating "noir" → attends to "black" (pos 2). The off-diagonal is the reordering signature.
Interpretability
Attention weights are the first form of interpretability in neural NLP.

"Why did the model predict X? It attended most to token Y."

Caveat: attention ≠ explanation. High weight ≠ causal importance (Jain & Wallace 2019). But useful for debugging.

Keras Functional API

Why Sequential fails. How to wire layers explicitly.

Why Sequential fails here

Sequential assumes a single linear chain: one input → one output. Encoder-decoder with attention has multiple inputs, multiple outputs, shared intermediate tensors. Sequential cannot express this.
Things Sequential cannot do
Encoder returns both all hidden states and the final state — two outputs.

Attention layer takes two inputs: all encoder states + current decoder state.

Decoder takes three inputs: previous token, context vector, previous hidden state.

At inference, encoder and decoder must run separately.
The Functional API pattern
Call layers like functions. Wire outputs explicitly: 
from tensorflow.keras import Model, Input
from tensorflow.keras.layers import LSTM, Embedding

enc_input = Input(shape=(None,))
enc_emb   = Embedding(V, D)(enc_input)

# return_sequences=True → all hidden states
# return_state=True     → final (h, c)
enc_out, h, c = LSTM(
    units,
    return_sequences=True,
    return_state=True
)(enc_emb)

encoder = Model(
    inputs=enc_input,
    outputs=[enc_out, h, c]
)

Key flags: return_sequences=True gives all hidden states (needed for attention). return_state=True also returns the final (h, c) pair. Both can be set simultaneously.

Teacher forcing

Problem during training: if step t produces a wrong token, step t+1 receives a wrong input — and the error compounds across the whole sequence.
Without teacher forcing
Step 1: "Le" ✓
Step 2: input = model's "Le" → predicts "chien" (wrong)
Step 3: input = "chien" → everything downstream is off

Error compounds. Training is slow and unstable.
With teacher forcing
Step 1: "Le" ✓
Step 2: input = ground-truth "Le" → predicts "chat" ✓
Step 3: input = ground-truth "chat" → predicts "noir" ✓

Clean gradient at every step. Fast and stable.
In Keras: pass the shifted target as decoder input — one forward pass covers the whole sequence simultaneously.
05

Limits and What's Next

Attention solved the bottleneck. A new one remains.

What attention didn't fix

What Bahdanau attention solved
Encoder bottleneck: decoder accesses all encoder states directly.

Long-range dependencies across the source: no longer lost.

Interpretability: attention maps as alignment for free.

Translation quality on long sentences: large improvement.
What it still didn't fix
Still sequential: encoder processes tokens left-to-right. Decoder generates one token at a time. Training cannot be parallelised.

Attention is cross-attention only: encoder tokens do not attend to each other. Within the encoder, information still flows only via LSTM.
The next step (S07): apply attention within the encoder itself — every token attends to every other token simultaneously. No LSTM. No sequentiality. Full parallelism. This is self-attention, the foundation of the Transformer.
