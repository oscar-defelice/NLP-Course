01

N-gram Language Models

Assigning probabilities to sequences of tokens.

What is a language model?

Definition. A language model is a probability distribution over sequences of tokens. Given a vocabulary $V$ and a sequence $W = (w_1, w_2, \ldots, w_T)$ with $w_t \in V$, a language model assigns: $$P(W) = P(w_1, w_2, \ldots, w_T) \in [0, 1]$$ subject to $\displaystyle\sum_{W \in V^*} P(W) = 1$, where $V^*$ is the set of all finite sequences over $V$.
Intuition
A language model answers: "How likely is it that a fluent speaker of this language would produce this sequence?"

  • "The dog barked loudly." — high probability
  • "Barked loudly the dog." — low probability (English)
  • "The dog meowed loudly." — medium (grammatical, unlikely)
Applications
  • Speech recognition: among all transcripts consistent with the audio signal, pick the most probable sentence
  • Machine translation: rank candidate translations by fluency
  • Text generation: sample new sequences from the distribution
  • Spell correction: $P(\text{"the cat"}) > P(\text{"teh cat"})$
  • Compression: a good LM = efficient code (Shannon 1951)

The chain rule decomposition

Any joint distribution over a sequence can be factored exactly using the chain rule of probability.

Chain rule: $$P(w_1, w_2, \ldots, w_T) = P(w_1)\cdot P(w_2 \mid w_1)\cdot P(w_3 \mid w_1, w_2) \cdots P(w_T \mid w_1, \ldots, w_{T-1})$$ $$= \prod_{t=1}^{T} P(w_t \mid w_1, \ldots, w_{t-1})$$
Worked example — 4-word sentence
$P(\text{"the cat sat down"})$
$= P(\text{the})$
$\times\; P(\text{cat} \mid \text{the})$
$\times\; P(\text{sat} \mid \text{the, cat})$
$\times\; P(\text{down} \mid \text{the, cat, sat})$

Each factor is a well-defined conditional probability.
The tractability problem
The last factor $P(w_T \mid w_1, \ldots, w_{T-1})$ conditions on a history of length $T-1$.

For a vocabulary of size $|V| = 50{,}000$ and sentences of length $T = 20$, the number of possible histories is $50{,}000^{19}$ — astronomically larger than any corpus.

We will never observe most histories even once. MLE is useless here.

The Markov assumption

Markov assumption of order $n-1$: the probability of a word depends only on the preceding $n-1$ words, not the full history. $$P(w_t \mid w_1, \ldots, w_{t-1}) \approx P(w_t \mid w_{t-n+1}, \ldots, w_{t-1})$$ This gives the n-gram model: $$P(w_1, \ldots, w_T) \approx \prod_{t=1}^{T} P(w_t \mid w_{t-n+1}, \ldots, w_{t-1})$$
Unigram — $n = 1$
Context window: none. $$P(w_t)$$ Words are fully independent. Equivalent to a bag-of-words model.

$P(\text{the cat sat}) = P(\text{the})\cdot P(\text{cat})\cdot P(\text{sat})$
Bigram — $n = 2$
Context window: 1 word. $$P(w_t \mid w_{t-1})$$ Captures immediate predecessor only.

$P(\text{the cat sat})$
$= P(\text{the})\cdot P(\text{cat}\mid\text{the})\cdot P(\text{sat}\mid\text{cat})$
Trigram — $n = 3$
Context window: 2 words. $$P(w_t \mid w_{t-2}, w_{t-1})$$ Captures two-word context.

$P(\text{sat}\mid\text{the, cat})$ captures "the cat ___" as a unit.

Maximum Likelihood Estimation — derivation

Given a corpus $\mathcal{D}$, we want the parameters $\theta$ that maximise $P(\mathcal{D} \mid \theta)$. For bigrams, the parameters are all conditional probabilities $P(w \mid u)$.

1
Log-likelihood of the corpus (chain rule over all sentences): $\mathcal{L}(\theta) = \sum_{(u,w) \in \mathcal{D}} C(u, w) \cdot \log P(w \mid u)$, where $C(u, w)$ is the count of bigram $(u, w)$ in $\mathcal{D}$.
2
Constraint: for each context $u$, $\displaystyle\sum_{w \in V} P(w \mid u) = 1$. Use a Lagrange multiplier $\lambda_u$.
3
Optimality condition: $\dfrac{\partial}{\partial P(w \mid u)}\left[\mathcal{L} - \lambda_u \sum_w P(w \mid u)\right] = 0$ gives $\dfrac{C(u, w)}{P(w \mid u)} = \lambda_u$.
4
Summing over $w$: $\lambda_u = C(u) \equiv \sum_w C(u, w)$.
MLE result: $$\hat{P}(w \mid u) = \frac{C(u, w)}{C(u)} = \frac{\text{count of bigram }(u, w)}{\text{count of unigram }u}$$ This is just the relative frequency — MLE recovers the empirical count ratio.

MLE — worked example

Corpus: "<s> the cat sat </s> <s> the cat ate </s> <s> the dog sat </s>"  (3 sentences, 9 content tokens)

Unigram counts $C(w)$
the: 3   cat: 2   sat: 2
ate: 1   dog: 1   <s>: 3
Bigram counts $C(u, w)$
<s>→the: 3   the→cat: 2   the→dog: 1
cat→sat: 1   cat→ate: 1   dog→sat: 1
Bigram probability table $\hat{P}(w \mid u)$

$\hat{P}(\text{the} \mid \langle s\rangle) = \dfrac{3}{3} = 1.00$

$\hat{P}(\text{cat} \mid \text{the}) = \dfrac{2}{3} \approx 0.67$

$\hat{P}(\text{dog} \mid \text{the}) = \dfrac{1}{3} \approx 0.33$

$\hat{P}(\text{sat} \mid \text{cat}) = \dfrac{1}{2} = 0.50$

$\hat{P}(\text{ate} \mid \text{cat}) = \dfrac{1}{2} = 0.50$

Check: $0.67 + 0.33 = 1$ ✓   $0.50 + 0.50 = 1$ ✓
$P(\text{"the cat sat"}) = P(\text{the}\mid\langle s\rangle)\cdot P(\text{cat}\mid\text{the})\cdot P(\text{sat}\mid\text{cat})\cdot P(\langle/s\rangle\mid\text{sat})$ $= 1.00 \times 0.67 \times 0.50 \times 1.00 = 0.333$

The sparsity problem

Zero-count problem. If a bigram $(u, w)$ never appears in the training corpus, MLE assigns $\hat{P}(w \mid u) = 0$. A single zero anywhere in the product kills the entire sentence probability: $$P(w_1 \cdots w_T) = \prod_t \hat{P}(w_t \mid w_{t-1}) = 0 \quad \text{if any factor is 0}$$
How bad is it in practice?
For a vocabulary of $|V| = 50{,}000$, the number of possible bigrams is $50{,}000^2 = 2.5 \times 10^9$.

A corpus of 1 billion words contains at most $10^9$ bigram tokens — fewer bigram types.

Most bigrams will never be observed, even in large corpora. For trigrams the situation is dramatically worse.

This is the curse of dimensionality applied to language.
Concrete example
Training corpus does not contain "ate the telescope".
Test sentence: "The astronomer ate the telescope cake."
→ $P = 0$. The model cannot evaluate this sentence at all.
Consequence: raw MLE is unusable for evaluation (perplexity = $\infty$) and generation (dead ends). We need smoothing.

Laplace (add-$\alpha$) smoothing

Idea: pretend we observed every possible n-gram at least $\alpha$ times (a pseudocount). $$\hat{P}_{\alpha}(w \mid u) = \frac{C(u, w) + \alpha}{C(u) + \alpha\,|V|}$$ where $|V|$ is the vocabulary size and $\alpha > 0$ (typically $\alpha = 1$ for add-one smoothing).
Properties
  • $\hat{P}_\alpha(w \mid u) > 0$ for all $w, u$ — no more zeros
  • $\displaystyle\sum_{w \in V} \hat{P}_\alpha(w \mid u) = 1$ — still a valid distribution
  • As $C(u, w) \to \infty$, $\hat{P}_\alpha \to \hat{P}_\text{MLE}$ — recovers MLE for frequent n-grams
  • As $\alpha \to 0$, recovers MLE; as $\alpha \to \infty$, approaches uniform distribution
Worked example ($|V| = 6$, $\alpha = 1$)
Same corpus as before. For the unseen bigram $(\text{dog, ate})$:
$\hat{P}_1(\text{ate}\mid\text{dog}) = \dfrac{0 + 1}{1 + 6} = \dfrac{1}{7} \approx 0.14$

For the seen bigram $(\text{the, cat})$:
$\hat{P}_1(\text{cat}\mid\text{the}) = \dfrac{2 + 1}{3 + 6} = \dfrac{3}{9} = 0.33$
(was 0.67 under MLE — smoothing shifts mass from frequent to rare)
Limitation: add-one smoothing is too aggressive for large vocabularies — it steals too much mass from observed n-grams. Better: Kneser-Ney smoothing (out of scope here).

Evaluating a language model — Perplexity

Given two models, how do we decide which is better? We need a scalar evaluation metric on held-out data.

Definition. The perplexity of a language model $P$ on a test sequence $w_1 \cdots w_N$ is: $$\text{PPL}(P, W) = P(w_1, \ldots, w_N)^{-1/N} = \exp\!\left(-\frac{1}{N}\sum_{t=1}^{N} \log P(w_t \mid w_{t-n+1}^{t-1})\right)$$
Intuition — branching factor
PPL = $k$ means the model is, on average, as uncertain at each step as if it had to choose uniformly from $k$ equally probable words.

A random model over a vocabulary of $|V|$ types has PPL $= |V|$.

A perfect model that knows the next word with certainty has PPL $= 1$.

Lower PPL = model assigns higher probability to the test data = better model.
Connection to cross-entropy
Cross-entropy of model $q$ with respect to true distribution $p$: $$H(p, q) = -\frac{1}{N}\sum_{t=1}^{N} \log_2 q(w_t \mid \text{ctx})$$ Then: $$\text{PPL} = 2^{H(p,q)}$$ PPL is just exponentiated cross-entropy (base 2). Minimising cross-entropy $\equiv$ minimising PPL.

This is also why NLL loss in neural LMs and PPL are the same objective.

Perplexity — worked example

Two bigram models evaluated on the test sentence: "the dog sat" ($N = 3$ predicted tokens + EOS).

Model A — trained on dogs
$P(\text{dog}\mid\text{the}) = 0.80$   $\log = -0.22$
$P(\text{sat}\mid\text{dog}) = 0.60$   $\log = -0.51$
$P(\langle/s\rangle\mid\text{sat}) = 0.90$   $\log = -0.11$

$\sum \log P = -0.84$
$\text{PPL}_A = \exp\!\left(\dfrac{0.84}{3}\right) = \exp(0.28) \approx \mathbf{1.32}$
Model B — trained on cats
$P(\text{dog}\mid\text{the}) = 0.05$   $\log = -3.00$
$P(\text{sat}\mid\text{dog}) = 0.10$   $\log = -2.30$
$P(\langle/s\rangle\mid\text{sat}) = 0.80$   $\log = -0.22$

$\sum \log P = -5.52$
$\text{PPL}_B = \exp\!\left(\dfrac{5.52}{3}\right) = \exp(1.84) \approx \mathbf{6.30}$
Model A has PPL $\approx 1.3$: it is barely surprised by "the dog sat". Effectively 1.3 equally plausible choices at each step.

Model B has PPL $\approx 6.3$: it is much more uncertain, as if choosing from ~6 options each time.
Benchmark values on AG News:
Unigram: PPL $\approx 1{,}500$–$2{,}500$
Bigram: PPL $\approx 300$–$500$
Trigram: PPL $\approx 150$–$250$
(Laplace smoothed, min_freq=2)

Text generation from an n-gram LM

A language model is not just an evaluator — it defines a generative process.

Greedy generation (argmax)
At each step $t$, pick the single most probable next word: $$w_t^* = \arg\max_{w \in V} P(w \mid w_{t-n+1}, \ldots, w_{t-1})$$ Fast, but often produces repetitive, generic output — it finds the mode, not diverse text.
Sampling with temperature $\tau$
Draw $w_t$ from: $$P_\tau(w \mid \text{ctx}) = \frac{\exp\!\left(\log P(w \mid \text{ctx}) / \tau\right)}{\sum_{w'} \exp\!\left(\log P(w' \mid \text{ctx}) / \tau\right)}$$ $\tau < 1$: sharper distribution (more deterministic)
$\tau = 1$: original distribution
$\tau > 1$: flatter distribution (more random)
Generation algorithm
1
Initialise context $= (\langle s\rangle, \ldots, \langle s\rangle)$ of length $n-1$.
2
Sample next token $w_t \sim P_\tau(\cdot \mid \text{context})$.
3
Append $w_t$ to output. Slide context window: drop oldest token, add $w_t$.
4
Stop when $\langle/s\rangle$ is sampled or max length is reached.
You will implement this in TP02 Part 5.

Limits of n-gram LMs

Fixed context window. By construction, an n-gram model cannot use any information from more than $n-1$ positions back. Long-range syntactic and semantic dependencies are invisible.
Example — subject-verb agreement
"The keys to the cabinet are on the table."

The subject is "keys" (plural), not "cabinet" (singular). A bigram or trigram model sees only "cabinet are" — it cannot resolve the agreement correctly.

GPT-3 handles this because its context is thousands of tokens long.
Sparsity vs context trade-off
Larger $n$ → longer context → better model quality
Larger $n$ → exponentially more n-gram types → worse coverage

In practice $n \leq 5$ for count-based models. Beyond that, almost every n-gram appears at most once — smoothing dominates.

Neural LMs (Session 5+) solve this by learning dense representations of context rather than counting.
N-gram LMs are still used in production ASR and MT systems as a rescoring component alongside neural models. They are fast, interpretable, and require no training hardware.

Naive Bayes

A generative classifier from Bayes' theorem.

Two families of classifiers

Generative models
Model the joint distribution $P(x, y) = P(x \mid y)\,P(y)$.

"How likely is this document to have been generated by class $y$?"

Naive Bayes is the canonical example.
Also: HMMs, LDAs, language models themselves.
Discriminative models
Model the conditional distribution $P(y \mid x)$ directly.

"What is the probability of label $y$ given this document?"

Logistic Regression is the canonical example.
Also: SVMs, neural networks, Transformers.
In practice: discriminative models tend to outperform generative ones when labelled data is plentiful. Generative models are useful when data is scarce or interpretability matters.

Bayes' theorem — the foundation

$$P(y \mid x) = \frac{P(x \mid y)\,P(y)}{P(x)}$$
posterior $\propto$ likelihood $\times$ prior
$P(y)$ — Prior
How frequent is class $y$ in the training set?
Estimated from label frequencies.
$P(x \mid y)$ — Likelihood
How likely is document $x$ given it belongs to class $y$?
This is where the "model" lives.
$P(x)$ — Evidence
Same for all classes. We can ignore it when comparing $P(y_1 \mid x)$ vs $P(y_2 \mid x)$.
Classification rule: $$\hat{y} = \arg\max_y P(y \mid x) = \arg\max_y\; P(x \mid y)\,P(y)$$

The naive independence assumption

Document $x$ is a sequence of words $(w_1, \ldots, w_T)$. Computing $P(x \mid y)$ exactly requires modelling all word interactions — intractable.

Naive Bayes assumption: given the class $y$, all words are conditionally independent. $$P(x \mid y) = \prod_{i=1}^{T} P(w_i \mid y)$$
Why naive?
Word order is completely ignored.
"dog bites man" = "man bites dog"

Strong correlations between words are ignored.
"New York" treated as two independent tokens.
Why does it still work?
Classification only needs to rank classes, not estimate exact probabilities.

Even with wrong probability estimates, the argmax is often correct.
Surprisingly competitive on text classification.

MLE for Naive Bayes

Two sets of parameters to estimate from the training data:

Class prior $\hat{P}(y)$
$$\hat{P}(y = c) = \frac{N_c}{N}$$ where $N_c$ = documents in class $c$, $N$ = total documents.
Word likelihood $\hat{P}(w \mid y)$
$$\hat{P}(w \mid y=c) = \frac{\text{count}(w, c)}{\sum_{w'} \text{count}(w', c)}$$
Number of times word $w$ appears in class $c$ docs, divided by total words in class $c$ docs.
Laplace smoothing for NB
Zero counts kill the product. Add-1 smoothing: $$\hat{P}(w \mid y=c) = \frac{\text{count}(w, c) + 1}{\sum_{w'} \text{count}(w', c) + |V|}$$
Same idea as for n-gram LMs. Every word gets at least one pseudocount.

Working in log-space

Numerical underflow. Multiplying many small probabilities: $$P(x \mid y) = \prod_{i=1}^{T} P(w_i \mid y)$$ For $T=100$ words with $P \approx 0.001$, this is $10^{-300}$ — underflows to 0 in float64.
Solution: work in log-space. $$\log P(y \mid x) \propto \log P(y) + \sum_{i=1}^{T} \log P(w_i \mid y)$$ Products become sums. No underflow. Classification rule unchanged (argmax is monotone).
Implementation pattern
log_prior = np.log(class_priors)           # shape (C,)
log_likelihood = X @ log_word_probs.T     # shape (N, C)  — bag of words dot log-probs
log_posterior = log_likelihood + log_prior
y_pred = np.argmax(log_posterior, axis=1)

Logistic Regression

A discriminative model. Model $P(y \mid x)$ directly.

From score to probability — the sigmoid

We want to output a probability in $[0, 1]$. Start with a linear score.

Linear score
$$z = \mathbf{w}^\top \mathbf{x} + b$$ $\mathbf{x} \in \mathbb{R}^V$ (bag-of-words or TF-IDF vector), $\mathbf{w} \in \mathbb{R}^V$ (weight vector), $b \in \mathbb{R}$ (bias).

$z$ is unbounded — it can be any real number.
Logistic (sigmoid) function
$$\sigma(z) = \frac{1}{1 + e^{-z}} \in (0, 1)$$ $$P(y=1 \mid \mathbf{x}) = \sigma(\mathbf{w}^\top \mathbf{x} + b)$$
Why sigmoid?
It is the unique function satisfying:
  • Maps $\mathbb{R} \to (0,1)$
  • Monotone (preserves ordering)
  • Differentiable everywhere
  • $\sigma(-z) = 1 - \sigma(z)$ (symmetric)

It also arises naturally from the log-odds formulation: $\log \frac{P(y=1)}{P(y=0)} = \mathbf{w}^\top \mathbf{x}$ implies $P(y=1) = \sigma(\mathbf{w}^\top \mathbf{x})$.

Multi-class — the softmax

For $C > 2$ classes, replace sigmoid with softmax.

$$P(y = c \mid \mathbf{x}) = \frac{e^{\mathbf{w}_c^\top \mathbf{x} + b_c}}{\displaystyle\sum_{c'=1}^{C} e^{\mathbf{w}_{c'}^\top \mathbf{x} + b_{c'}}}$$
One weight vector $\mathbf{w}_c$ per class. Outputs sum to 1.
AG News: 4 classes
$\mathbf{W} \in \mathbb{R}^{4 \times V}$ — one row per class.
$\mathbf{z} = \mathbf{W}\mathbf{x} + \mathbf{b} \in \mathbb{R}^4$
$\hat{\mathbf{p}} = \text{softmax}(\mathbf{z}) \in \mathbb{R}^4$
$\hat{y} = \arg\max_c \hat{p}_c$
Numerical stability
Never compute $e^z$ directly — overflow.
Subtract the max first:
$\text{softmax}(\mathbf{z})_c = \frac{e^{z_c - \max \mathbf{z}}}{\sum_{c'} e^{z_{c'} - \max \mathbf{z}}}$
This is what scipy/PyTorch do internally.

The loss function — cross-entropy

We need a scalar to minimise. It must penalise wrong or uncertain predictions.

Negative log-likelihood (= cross-entropy loss) for one example: $$\mathcal{L}(\mathbf{w}) = -\log P(y^* \mid \mathbf{x}) = -\log \hat{p}_{y^*}$$ where $y^*$ is the true label.
Why NLL?
It is the maximum likelihood criterion: maximising $\prod_i P(y_i \mid \mathbf{x}_i)$ over the dataset is equivalent to minimising $-\sum_i \log P(y_i \mid \mathbf{x}_i)$.

Also: penalises confident wrong predictions very heavily. $-\log(0.01) \approx 4.6$ vs $-\log(0.99) \approx 0.01$.
Batch loss
$$\mathcal{L} = -\frac{1}{N}\sum_{i=1}^{N} \log P(y_i \mid \mathbf{x}_i)$$ Average over the training set. Add L2 regularisation to avoid overfitting: $$\mathcal{L}_{\text{reg}} = \mathcal{L} + \frac{\lambda}{2}\|\mathbf{w}\|^2$$

Gradient derivation — binary case

Derive $\nabla_{\mathbf{w}} \mathcal{L}$ for binary LR. The result is elegant.

1
Loss for one example: $\mathcal{L} = -[y \log \hat{p} + (1-y)\log(1-\hat{p})]$,   $\hat{p} = \sigma(z)$,   $z = \mathbf{w}^\top \mathbf{x}$
2
Chain rule: $\dfrac{\partial \mathcal{L}}{\partial \mathbf{w}} = \dfrac{\partial \mathcal{L}}{\partial \hat{p}} \cdot \dfrac{\partial \hat{p}}{\partial z} \cdot \dfrac{\partial z}{\partial \mathbf{w}}$
3
$\dfrac{\partial \mathcal{L}}{\partial \hat{p}} = -\dfrac{y}{\hat{p}} + \dfrac{1-y}{1-\hat{p}}$    $\dfrac{\partial \hat{p}}{\partial z} = \sigma(z)(1-\sigma(z)) = \hat{p}(1-\hat{p})$    $\dfrac{\partial z}{\partial \mathbf{w}} = \mathbf{x}$
4
Multiplying out: $\dfrac{\partial \mathcal{L}}{\partial \mathbf{w}} = (\hat{p} - y)\,\mathbf{x}$
Final result: $$\nabla_{\mathbf{w}} \mathcal{L} = (\hat{p} - y)\,\mathbf{x}$$ The gradient is simply the prediction error scaled by the input. This is why logistic regression is so well-behaved numerically.

Gradient descent

Update rule (batch gradient descent): $$\mathbf{w} \leftarrow \mathbf{w} - \eta \cdot \frac{1}{N}\sum_{i=1}^{N} (\hat{p}_i - y_i)\,\mathbf{x}_i$$ where $\eta > 0$ is the learning rate.
Batch GD
Use all $N$ examples per update.
Stable. Slow for large datasets.
Stochastic GD (SGD)
One example per update.
Noisy. Fast. Can escape local minima.
Mini-batch SGD
$B$ examples per update.
Best of both. Standard in practice.
Also what sklearn does internally.
In practice: sklearn.linear_model.LogisticRegression uses L-BFGS by default — a second-order optimiser. You do not need to implement GD yourself, but you must understand what it is minimising and why.

NB vs Logistic Regression — comparison

Property Naive Bayes Logistic Regression
Type Generative Discriminative
What is modelled $P(x, y) = P(x\mid y)\,P(y)$ $P(y \mid x)$ directly
Independence assumption Yes — features are independent given class No
Training Count and normalise — closed form Gradient descent — iterative
Small data Good — fewer parameters to estimate Worse — more prone to overfitting
Large data Plateaus Keeps improving
Interpretability High — word likelihoods are meaningful Moderate — weights indicate feature importance
For AG News, expect NB to achieve ~88-90% accuracy, LR ~91-93%. Both crush the majority baseline (~25%).
