00

The last bottleneck

Attention fixed the fixed-vector problem. Recurrence itself is still a problem.

What attention did not fix

Session 06 added attention to the LSTM seq2seq model. The decoder can now look at all encoder states. But the encoder itself is still a recurrent network — and recurrence has a fundamental cost.
The recurrence bottleneck
To compute h_t you must first have h_{t-1}.
Steps cannot be parallelised across the sequence.

For a document of 512 tokens: 512 sequential LSTM steps. Modern GPUs are built for massive parallelism — the sequential dependency leaves most hardware idle.

Training BERT-scale models on LSTMs would take months. Transformers train in days because they parallelise completely.
The Transformer insight (Vaswani et al., 2017)
"Attention is all you need."

Remove the RNN entirely.
Compute attention between every pair of positions simultaneously.
All positions are processed in parallel.

h_i can attend to h_j for any j — no decay over distance, no vanishing gradient across the sequence.
Key difference from S06 attention: In S06, attention was between decoder states and encoder states (cross-attention). In the Transformer encoder, each position attends to every other position in the same sequence — this is self-attention.
01

Self-Attention

Query · Key · Value · Every token attends to every other token simultaneously.

Self-attention — the intuition

In the sentence "The animal didn't cross the street because it was too tired", what does it refer to? animal or street? Self-attention lets the model look at the whole sentence simultaneously and build a representation of it that incorporates context from animal.
The database analogy
Think of self-attention as a soft database query:

Query Q: what am I looking for?
Key K: what does each entry offer?
Value V: what does each entry contain?

For each position, Q is compared against all K's to get relevance scores. The output is a weighted sum of V's — a soft retrieval.
The formulas
For a sequence of n tokens, embed to matrix X (n × d).

Project to Q, K, V with learned weight matrices:
Q = X·W_Q K = X·W_K V = X·W_V
Compute attention:
Attention(Q,K,V) = softmax( Q·K^T / √d_k ) · V
√d_k: scaling factor to prevent large dot products saturating softmax.
Parallelism
Q·K^T is a single matrix multiplication: n×d times d×n = n×n.
All n² attention scores are computed simultaneously.
Then softmax, then another matrix multiply with V.

No loop, no sequential dependency. The entire self-attention for a sequence of 512 tokens is three matrix multiplications — fully parallelisable on GPU.
Quadratic cost
The n×n attention matrix is O(n²) in memory and compute. For 512 tokens: 262K entries. For 4K tokens: 16M entries.

This is why standard BERT is limited to 512 tokens. Efficient Transformers (Longformer, BigBird) use sparse attention patterns to scale.

Self-attention — one head visualised

Each token computes Q, K, V. Every token attends to every other. Output is a weighted sum of all V's.

02

Transformer Architecture

Stacking self-attention blocks. Positional encoding. Feed-forward layers.

The Transformer encoder block

Multi-Head Self-Attention
Run H parallel attention heads, each with its own W_Q, W_K, W_V.
Concatenate outputs, project back to model dimension.

Multiple heads allow the model to attend to different aspects simultaneously: one head for syntax, one for coreference, one for semantics.
Positional Encoding
Self-attention is permutation-invariant — it does not know token order.
Add a sinusoidal position vector to each embedding:

PE(pos, 2i) = sin(pos / 10000^{2i/d})
PE(pos, 2i+1) = cos(pos / 10000^{2i/d})
Each position gets a unique pattern. The model learns to use it.
Residual connections + LayerNorm
output = LayerNorm(x + Sublayer(x))

Residuals make gradients flow directly to earlier layers. LayerNorm stabilises training. Without them, very deep Transformers (12+ layers) do not converge.
03

BERT and Pre-training

Learning from unlabelled text at massive scale.

From Transformer to BERT

Vaswani et al. (2017) introduced the Transformer for translation (encoder + decoder). Devlin et al. (2018) took only the encoder stack and pre-trained it on 3.3 billion words of unlabelled text — this is BERT.
Architecture
BERT-base:
12 Transformer encoder blocks
12 attention heads
Hidden size 768
110M parameters

BERT-large: 24 blocks, 1024 hidden, 340M params. DistilBERT: 6 blocks, 66M params, 97% of BERT performance.
Pre-training task 1: MLM
Masked Language Modelling:
Randomly mask 15% of tokens.
Predict the masked token from context.

Input: "The [MASK] sat on the mat"
Target: predict "cat"

Forces the model to build deep bidirectional context representations.
Pre-training task 2: NSP
Next Sentence Prediction:
Given two sentences A and B:
Is B the actual next sentence after A?

50% of pairs: consecutive (positive).
50% of pairs: random (negative).

Teaches the model sentence-level relationships.
The key insight: pre-training on unlabelled text is free and scales arbitrarily. BERT was trained on Wikipedia + BookCorpus for 4 days on 64 TPU chips. You inherit all that knowledge and add a thin classification head on top.

Contextual embeddings — why BERT beats GloVe

GloVe — static embeddings
"bank" has one vector regardless of context.

"I went to the river bank" → same vector as
"I deposited money at the bank"

The embedding is a property of the word, not the sentence. Polysemy is unresolvable.
LSTM — partially contextual
"bank" in position i is influenced by tokens 1 to i (and 1 to i from the right in bidirectional LSTM).

Better than GloVe, but still trained on one task at a time. No transfer from the rich signal of 3B words of general text.
BERT — deeply contextual
"bank" has a different vector in every sentence it appears in.

"I went to the river bank": the 12 self-attention layers integrate signals from river, went, and every other token bidirectionally. The representation knows it is a geographic feature.

"I deposited money at the bank": integrates deposited, money. The representation knows it is a financial institution.

Same word, completely different vectors. This is contextualisation.
04

Fine-tuning DistilBERT

Transfer learning at scale. One pre-trained model, many tasks.

Fine-tuning paradigm

Fine-tuning: start from a pre-trained model and continue training on your labelled task data. The pre-trained weights are updated (all or partially).
Pre-trained model
DistilBERT: 66M parameters,
pre-trained on 3.3B words.
Already encodes:
— syntax and grammar
— entity recognition
— coreference
— semantic similarity
— world knowledge

Downloaded in seconds from HuggingFace Hub.
Classification head
Add a thin head on top of the [CLS] token output:

[CLS] → Dropout → Dense(2, softmax)

The [CLS] token is prepended to every input. After 6 Transformer blocks it accumulates a summary of the whole sequence. This is what you classify on.
Training
SST-2 dataset: 67K movie reviews, binary sentiment.

Fine-tune with small learning rate (2e-5 to 5e-5).
2–3 epochs is usually enough.
Colab free GPU: ~15 min.

Typical result: 91–93% accuracy on SST-2 val set.
Compare: LR + TF-IDF on SST-2 ≈ 85–87%.

DistilBERT fine-tuning — code

from transformers import (
    DistilBertTokenizerFast,
    TFDistilBertForSequenceClassification,
)
from datasets import load_dataset
import tensorflow as tf

# ── Load dataset ──────────────────────
sst2 = load_dataset("stanfordnlp/sst2")

# ── Tokenise ──────────────────────────
tokenizer = DistilBertTokenizerFast.from_pretrained(
    "distilbert-base-uncased"
)

def tokenise(batch):
    return tokenizer(
        batch["sentence"],
        truncation=True,
        padding="max_length",
        max_length=128,
    )

sst2_enc = sst2.map(tokenise, batched=True)
# ── Model ─────────────────────────────
model = TFDistilBertForSequenceClassification.from_pretrained(
    "distilbert-base-uncased",
    num_labels=2,
)

model.compile(
    optimizer=tf.keras.optimizers.Adam(2e-5),
    loss=tf.keras.losses.SparseCategoricalCrossentropy(
        from_logits=True
    ),
    metrics=["accuracy"],
)

What's Left — and What's Next

Limits. GPT vs BERT. The landscape beyond this course.

BERT vs GPT — two flavours of Transformer

BERT — encoder only, bidirectional
Reads the whole sequence at once.
Each token attends to all other tokens (left + right).
Pre-training: masked language modelling (fill in the blank).

Good for: classification, NER, QA, anything that reads and understands.
Cannot: generate text (no left-to-right decoder).
GPT — decoder only, causal
Generates text left to right.
Each token attends only to previous tokens (causal mask).
Pre-training: next token prediction (language modelling).

Good for: text generation, dialogue, code, instruction following.
Less good for: classification tasks where bidirectionality matters.
The limits of this course's models
Even fine-tuned DistilBERT:
— 512 token context window
— No document-level reasoning
— No tool use or retrieval
— No multilinguality (base model)
— Hallucination: does not apply (discriminative), but classification on out-of-distribution inputs fails silently

Modern LLMs (GPT-4, Claude, Gemini) address many of these, but the core attention mechanism is identical to what you built in TP-06.
The core insight you now have:
Transformers = stacked self-attention + feed-forward + residuals + LayerNorm.
Scale up: 12 → 96 blocks, 768 → 12288 hidden, billions of parameters.
The architecture does not change. The capability emerges from scale.
